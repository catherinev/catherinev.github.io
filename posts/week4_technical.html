#map in Ruby

The #map method in Ruby can used to transform arrays, hashes, and ranges by calling a block of code on each element and pushing each result to a new array. So, regardless of the class of the input (array, hash, or range), the output is a new array.

input.map { |item| block } => output_array

You can interpret this syntax as follows.

  • input is an array, hash, or range.
  • item represents the elements in input that are being iterated over. In an array/range, item represents each element; in a hash, item represents the key/value pair.
  • block is the block of code that will be performed on each element in input. The result of this block is pushed to the output_array.
  • output_array is an array that contains the values returned by the block for each of the elements in input.

Let's look at some examples. Suppose you have an array of numbers and want to convert them all to strings. You can use #map:

[1,2,3,4,5].map {|element| element.to_s} => ["1", "2", "3", "4", "5"]

Or perhaps you'd like to do something to each key or value in a hash. See the examples below:

{"first" => 1, "second" => 2, "third" = >3}].map {|key, value| value + 2} => [3, 4, 5]

{"first" => 1, "second" => 2, "third" => 3}].map {|key, value| key.capitalize} => ["First, "Second", "Third"]

Finally, #map can be called on ranges, as shown in the example below.

(1..8).map {|number| number * 4} => [4, 8, 12, 16, 20, 24, 28, 32]

#map is useful if you are trying to do the same thing to each of the elements in a particular collection and want the result to be an array. Consider how #map differs from #each.

  • #each calls a block on each element but returns the original array.
  • #map calls a block on each element but also pushes the result of the block in a new array and returns the resulting array.
