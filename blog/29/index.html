Using LyX to Display Code: Part 2

Set up minted in LaTeX
Note: part 1 of this tutorial can be found here.

0. Install minted on your computer

If you have TeXLive 2014 (included in the MacTeX installation I recommended), minted is already installed on your machine.

If not, see the instructions at this link: installation instructions.

1. Make a sample LaTeX file using minted

Open TeXShop and create a new LaTeX file. Copy-paste the following code into your empty LaTeX document.

\documentclass[11pt, oneside]{article}
\usepackage{geometry}

\geometry{letterpaper}
\usepackage{graphicx}
\usepackage{amssymb}
\usepackage{minted}

\usepackage{minted}
\definecolor{bg}{rgb}{0.95,0.95,0.95}
\newminted{python}{bgcolor=bg}

\title{Is minted working?}

\begin{document}
\maketitle
\begin{pythoncode}
import random

def say_hello(name):
    print "hello %s" % name

\end{pythoncode}

\end{document}

In the terminal, navigate to the folder you have created the LaTeX document in. Then run the following command, replacing <your-filename> with the name of your file, including the .tex extension:

pdflatex -shell-escape <your-filename>

# for example:
pdflatex -shell-escape test_file.tex

If this doesn't throw any errors, you are good to go! That means that minted is working fine on your computer, and you can skip ahead to part 3 of this tutorial. If you'd like to see the output, open the PDF file that was created (your-filename.pdf).

What to do if you got error messages

As always, when you get an error message you must start by reading the error message. Did you copy-paste the code correctly? Is there a bracket missing somewhere? In essence, start by looking for errors that are easy to make (and easy to correct).

If there are no syntax errors, then double check you followed exactly the procedure I outlined. Did you run the pdflatex command from the command-line? You CANNOT just click "Typeset" from within TeXShop because of the extra flag -shell-escape. This is really important and minted will not compile without the flag.

If there are no syntax errors and you ran the correct command in the terminal, then return to the error messgage. What does it say? If there is a problem with minted, that may mean that you don't have it installed correctly on your machine. If there is a problem with a different LaTeX package (e.g. ifplatform or ifthen), then chances are that package isn't installed correctly on your machine. In this case, I implore you to download TeX Live 2014 (which is included in MacTeX). It is by far the easiest solution, although it may take some time to download and will take up almost 5 GB of space on your computer. However, if you really want to troubleshoot without installing TeX Live 2014, here are some resources:

  • minted documentation
  • Installing new LaTeX packages

It is also possible there is an issue with Pygments or Python, in which case you should make sure you have up-to-date versions of both.

A brief explanation of what we did

Note: It is not necessary for you to understand this section to continue the tutorial.

If you are familiar with LaTeX, the only part of the given code that may be unfamiliar is \newminted{python}{bgcolor=bg}. This is minted's way of defining shortcuts. Without this command, you can use minted as follows: \begin{minted}{python}
print "hello world"
\end{minted}
 \newminted{python}{bgcolor=bg} creates an alias for the minted environment.

More details, including other options minted provides, can be found in its documentation.
