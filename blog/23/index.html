My First Rails App!

During phase 2, we used a framework called Sinatra, which is a lightweight version of Rails. Since it doesn't handle as many of the details as Rails does, it's a great way to get familiar with what is happening behind the scenes before jumping into Rails.

I'm starting Phase 3 next week, so in preparation we learned Rails this weekend. The core assignment was to build a blog application. This is what I ended up with: The Awesome Blogger. If you want to see a blog without logging in, check out this link. You'll find the posts to be thrilling. If you are interested in the source code for the site, you can find it here.

The app is based on a blogger app I built in Sinatra during phase 2 (which explains the award-worthy design and color scheme ). To build it, I relied heavily on the Getting Started with Rails guide and Michael Hartl's Ruby on Rails Tutorial as references, both of which I found to be very helpful.

There are many aspects of this app that could be better, perhaps most obviously the design. Ajax would make comment-posting a better experience, as well as various other actions on the site. Also, at the moment anyone can post a comment; ultimately, it would probably be better if a user had to be signed in to leave a comment. Paginating would also be great.

There are still some lingering issues. For example, my error messages work just fine in development but aren't working in production. Hmmm. Also, I did not use TDD. That's my goal for my next app -- I'm pretty sure we're going to get into TDD this week.

Still, I'm pretty happy with it as a first project!

Note: I am treating this app as a toy. You are welcome to play with it, but I may drop the database as I continue to tinker around. That means anything you create has a chance of being lost. Also, it's currently being hosted on Heroku with one dyno, which means it may take a little while to fire up after you click the link. If you find any bugs, let me know!
