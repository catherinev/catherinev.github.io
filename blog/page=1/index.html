Using LyX to Display Code: Part 3

Set up a custom environment in LyX
Note: part 2 of this tutorial can be found here.

0. Make sure minted is working in LaTeX and familiarize yourself with LyX

In other words, start by doing part 2 of this tutorial. Then, if you have not already used LyX, you should probably play around with it for a bit. See what it can do. Use different environments; generate a PDF of your document; change the font style; insert some math; maybe even insert a table (and see if you can customize it).

1. Find the LyX source code files we are going to modify

We are going to add a new environment to the stdlayouts.inc file. On a Mac, this file can be found at the following path: /Applications/LyX.app/Contents/Resources/layouts. From the Finder, you can access this by first opening the Applications folder, then right clicking LyX and selecting "Show Package Contents". From there, select Contents then Resources then layouts, and you should find the desired file.   read more »

As I mentioned in the intro, I have a Mac, so this tutorial will be specifically aimed at Mac OS X. However, it may not be too hard to modify it for Linux (and possibly Windows), especially if you are familiar with LaTeX and how it is configured on your machine.

Using LyX to Display Code: Intro

What is LyX?

Before we talk about LyX, let's talk briefly about LaTeX, which is a widely used markup language that is especially popular in math and the sciences. I first fell in love with LaTeX when I was writing my math thesis in college. Then I went to math grad school and wrote basically everything in TeX. It just makes things look so pretty.

Anyways, a few years ago a friend of mine introduced me to LyX. LyX is WYSIWYM interface for LaTeX. Even though LyX code still needs to be compiled, its interface reflects the compiled document much more closely than outright code does. Don't Park There Chicago: the design

See this post for more background on the Don't Park There Chicago

The overall design of Don't Park There Chicago went through 3 distinct phases. The first looked like this:

Design 1

It's pretty snazzy looking. However, it had a few problems. The first was that the background picture was a large file that took a while to load. We could probably have decreased the size of the file, but there was a bigger problem: the design impeded the functionality of the site. At the bottom of the page was a super cool map and more information about street sweeping -- information that is valuable to many users. Users must either scroll down or click the down arrow button to get that information. But user testing revealed that almost no one scrolled or clicked the button.
