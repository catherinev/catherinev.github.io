Using LyX to Display Code: Intro

What is LyX?

Before we talk about LyX, let's talk briefly about LaTeX, which is a widely used markup language that is especially popular in math and the sciences. I first fell in love with LaTeX when I was writing my math thesis in college. Then I went to math grad school and wrote basically everything in TeX. It just makes things look so pretty.

Anyways, a few years ago a friend of mine introduced me to LyX. LyX is WYSIWYM interface for LaTeX. Even though LyX code still needs to be compiled, its interface reflects the compiled document much more closely than outright code does. To be perfectly honest, I was happy with LaTeX just the way it was; I don't mind the fact that it's not WYSIWYG. But it has a few distinct downsides:

  • LaTeX is compiled, which means you have to recompile to see any formatting changes. Most of the time, this isn't a big deal. But sometimes a document takes a little while to compile, which is annoying.
  • Tables. To me, tables are one of the most annoying things to work with in any sort of programming/typesetting language. They are distinctly unreadable until they are compiled.
  • Page breaks. Who knows where those will be.
  • Math. In LaTeX, math equations look a bit like gibberish (although you do learn to read that gibberish, to some degree).
  • There is a lot of code you don't need to see all the time. For example, once you've formatted some text, you don't need to see the code that controls the formatting. This makes the document difficult to read and even more difficult to scan.
  • It's got its own language. This didn't matter to me since I already knew the language, but it's something to consider for those of you who don't already know LaTeX.

LyX solves most of these problems (I still can't tell where the page breaks are going to be). Most importantly, it makes uncompiled documents much more readable, which is important for documents that I'm creating for myself. Big difference between that and something I'm going to be publishing. If it's just for me, that means it's probably some sort of reference, and I put pretty high value on being able to peruse it quickly.

Bottom line: there is a place for LyX in my life.

Why is LyX relevant to people who code?

It might not be. On the other hand, it might be. Any type of document that would be a pain to write with Microsoft Office is probably pretty easy to write with LyX/LaTeX. For me, that includes any document with math and also many documents that include code.

The Problem

I know Ruby pretty well and am in the process of learning Python. Yesterday, I decided it would be a useful exercise to create a table comparing Ruby to Python. Since tables are much easier to implement in LyX than in LaTeX, I opened a LyX document and got started.

I knew this document was going to have code samples in it, and pretty quickly I decided I wanted the code to look nice. Specifically, I wanted the code to look a lot like it does in my text editor, where keywords are highlighted in different colors.

The solution

LyX is open source, which means I can actually edit the source code. This is glorious; it means that if I don't like some aspect of LyX or think of some feature I'd like it to have, I can change it. In other words, I can add a custom environment that automatically formats Python code.

So, the solution we are after is the following: To create a custom environment in LyX that displays code prettily. What does that mean? LaTeX has environments such as section, subsection, quote, itemize, title, etc. Each environment has predefined style settings that can be adjusted (if you are familiar with HTML/CSS, this should sound very familiar).
