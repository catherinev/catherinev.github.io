Classes in Ruby

Classesare a natural way to group similar types of objects in Ruby. For example, Array is a class. There are two ways to define a new object in the class Array : arr_1 = Array.new
arr_1[0] = "Hello"
arr_1[1] = "world"
arr_2 = [1, 2, 3, 4] arr_1 has been defined using constructor notation and arr_2 has been defined using literal notation, but both are the same type of object. arr_1 and arr_2 are called instancesof the class Array. So a class is a category, and an instance is a particular object that falls into that category. Other examples of classes are String, Fixnum, Float, and Hash.

So why are classes helpful? Because we often want to know similar things or perform similar tasks on objects in the same class. Each class has associated methodsthat do something to an instance of the class. For example, #length, #each, and #sort are all methods associated with the class Array. Similarly, #length, #downcase, and #each_char are all methods associated with the class String. Each of these methods can be called on any instance of the associated class. (Note that some methods are associated with more than one class.)

All the above examples of classes come with Ruby. But it is also possible to define your own classes and associated methods. It is a good idea to create a new class of objects any time you think you will be creating multiple instancesof that object and may want to perform the same tasks (i.e. call the same method) on those instances.

For example, you may want to define the class Rectangle :

class Rectangle
  def initialize(width, height)
    @width = width
    @height = height
  end

  def perimeter
    2 * @width + 2 * @height
  end

  def area
    @width * @height
  end
end

rec = Rectangle.new(3, 4)

Above, rec is an instance of the class Rectangle.

It is natural to create a class for Rectangle because there are many possible instances of the class that can now all use the same methods.

The first method we have is #initialize. This method is run automatically every time you create a new instance of the class. In our Rectangle example, all that the initialize method is doing is defining the instance variables @width and @height. An instance variable is a variable associated with an instance of the object and always begins with @. Instance variables can be used in any method within the class in which they are defined. In rec, @width = 3 and @height = 4 , so any Rectangle method that we call on rec will use the values 3 and 4 for @width and @height , respectively.

After #initialize , we defined methods to find the perimeter and area of a rectangle. They can be called as follows:

rec.perimeter => 14
rec.area => 12

This is a brief introduction to classes set in the context of a fairly simple example, but hopefully it gives you a sense of what classes are and when they should be used.
